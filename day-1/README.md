# День 1 — Переменные и связывание (Binding)

> **Ключевая концепция:** В Rust `let` создаёт не "переменную", а **связывание** (binding) — привязку имени к значению. Это фундаментальное отличие от других языков.

---

## Содержимое каталога

| Файл | Описание |
|------|----------|

## Быстрый старт

### 1. Просмотр файлов

```bash
# Список файлов в каталоге
ls -la day-1/

# Просмотр содержимого файла
cat day-1/variables.rs

# Или постранично
less day-1/variables.rs
```

### 2. Создание проекта для запуска

```bash
# Создать новый Rust-проект
cargo new rust-day1 --bin

# Скопировать упражнения как main.rs
cp day-1/variables_exercise.rs rust-day1/src/main.rs

# Перейти в проект
cd rust-day1
```

### 3. Запуск и тестирование

```bash
# Запустить программу
cargo run

# Запустить все тесты
cargo test

# Запустить конкретный тест
cargo test t_mutability

# Тесты с выводом println!
cargo test -- --nocapture
```

## Полезные команды Cargo

### Создание проектов

| Команда | Описание |
|---------|----------|
| `cargo new name --bin` | Создать бинарный проект (исполняемый файл) |
| `cargo new name --lib` | Создать библиотеку |
| `cargo init` | Инициализировать проект в текущей папке |
| `cargo init --lib` | Инициализировать библиотеку в текущей папке |

### Сборка и запуск

| Команда | Описание |
|---------|----------|
| `cargo build` | Скомпилировать (debug-режим) |
| `cargo build --release` | Скомпилировать с оптимизациями |
| `cargo run` | Скомпилировать и запустить |
| `cargo run --release` | Запустить оптимизированную версию |
| `cargo check` | Быстрая проверка без генерации бинарника |

### Тестирование

| Команда | Описание |
|---------|----------|
| `cargo test` | Запустить все тесты |
| `cargo test name` | Запустить тесты, содержащие `name` |
| `cargo test -- --nocapture` | Тесты с выводом `println!` |
| `cargo test -- --test-threads=1` | Тесты последовательно (не параллельно) |
| `cargo bench` | Запустить бенчмарки |

### Качество кода

| Команда | Описание |
|---------|----------|
| `cargo fmt` | Отформатировать код |
| `cargo fmt --check` | Проверить форматирование (без изменений) |
| `cargo clippy` | Линтер — найти проблемы в коде |
| `cargo clippy -- -D warnings` | Clippy: warnings как ошибки |

### Зависимости

| Команда | Описание |
|---------|----------|
| `cargo add crate_name` | Добавить зависимость |
| `cargo add crate_name@1.2.3` | Добавить конкретную версию |
| `cargo remove crate_name` | Удалить зависимость |
| `cargo update` | Обновить зависимости |
| `cargo tree` | Показать дерево зависимостей |

### Документация и информация

| Команда | Описание |
|---------|----------|
| `cargo doc` | Сгенерировать документацию |
| `cargo doc --open` | Сгенерировать и открыть в браузере |
| `cargo search crate_name` | Поиск крейтов на crates.io |
| `cargo info crate_name` | Информация о крейте |

### Публикация

| Команда | Описание |
|---------|----------|
| `cargo login` | Авторизация на crates.io |
| `cargo publish` | Опубликовать крейт |
| `cargo publish --dry-run` | Проверить публикацию без отправки |

### Очистка

| Команда | Описание |
|---------|----------|
| `cargo clean` | Удалить папку `target/` |
